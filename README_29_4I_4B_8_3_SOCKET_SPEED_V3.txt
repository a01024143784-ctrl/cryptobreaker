29-4I-4B-8-3 SOCKET SPEED V3

적용 내용
1) SSE 최신 프레임 유지
- 브라우저 큐가 밀리면 오래된 프레임을 버리고 최신 데이터만 유지
- 렌더 지연/밀림 현상 완화

2) 대용량 payload 분리
- all_coins 전체 데이터는 2초 단위로만 전송
- 평소에는 TOP/급등급락/OI 중심 light payload 전송
- 네트워크/브라우저 JSON 파싱 부담 감소

3) 프론트 렌더 분산
- 현재 보고 있는 페이지를 우선 갱신
- 보조 카드/즐겨찾기/고래센터 렌더는 idle/timeout으로 분산
- 중복 데이터는 signature 비교로 렌더 생략

4) 백그라운드 절전
- 탭 비활성화 시 무거운 렌더 스킵
- 애니메이션 일시정지 클래스 적용

체감 목표
- 데이터 밀림 감소
- 화면 끊김 감소
- CPU/GPU 사용량 감소
- 방송 중 장시간 켜놔도 안정성 향상
